Available browser endpoints

/setup

Loads an user interface which allows modification of the stored audd.io api key

/listen

Activates microphone on Raspberry Pi to record 10 seconds of audio, sends it to the online API for parsing and returns the response in a HTML friendly format. Shows status pages along the way so the user knows something is happening.  (Uses one audd.io credit)

/listen/silent

Activates microphone on Raspberry Pi to record 10 seconds of audio, sends it to the online API for parsing and returns the response in a HTML friendly format. Does not show status pages along the way, just loads for a long time with recording and processing.  (Uses one audd.io credit)

/display

Displays output of the most recently parsed track (Does not use an audd.io credit - all run locally)

/art

Redirects to the artwork URL generated following /listen or /listen/silent

Available API endpoints

/listen/json

Activates microphone on Raspberry Pi to record 10 seconds of audio, sends it to the online API for parsing and returns the response in JSON. This can take 10-20 seconds to return anything as it needs to listen and then parse via the web API.  (Uses one audd.io credit)

/display/json

Displays output of the most recently parsed track in JSON format (Does not use an audd.io credit - all run locally)

/download_art

Downloads the artwork for the most recently parsed track to the local folder as artwork.jpeg. Returns JSON showing the full path (Does not use an audd.io credit)
